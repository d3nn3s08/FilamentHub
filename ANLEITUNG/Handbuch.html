🧵 FilamentHub — Handbuch

Beta v1.6 · Vollständige Dokumentation auf Deutsch

FilamentHub ist ein zentrales Verwaltungssystem für 3D-Drucker-Filament. Es verbindet sich mit Bambu Lab Druckern (MQTT + Cloud), Klipper/Moonraker und Happy Hare MMU und verfolgt automatisch Spulen, Verbrauch und Druckjobs.

📦 Installation

Docker (Unraid / NAS) · Windows lokal

Docker auf Unraid

⚠️ Pflicht: network_mode: host muss gesetzt sein — sonst findet der Drucker-Scanner das LAN nicht!
  1. Datei docker-compose.yml auf Unraid: /mnt/user/appdata/filamenthub/
  2. cp .env.example .env — Admin-Passwort-Hash generieren und eintragen
  3. Image bauen: docker build -t filamenthub:latest .
  4. Starten: docker compose up -d
  5. Öffnen: http://UNRAID-IP:8085
# Admin-Passwort Hash generieren:
python3 -c "import bcrypt; print(bcrypt.hashpw(b'DEIN_PASSWORT', bcrypt.gensalt()).decode())"

Docker auf NAS (QNAP)

Die Datei nas-stable/docker-compose.yml ist speziell für QNAP vorbereitet.

# Via SSH auf dem NAS:
cd "/share/backup-archive/Denis Programme/FilamentHub/nas-stable"
docker build -t filamenthub:latest .
docker compose up -d

# Nach einem Update:
bash rebuild.sh
Synology DSM: Container Manager GUI unterstützt network_mode: host nicht. Muss per SSH gesetzt werden. Ohne host-Modus findet der Scanner keine Drucker — Drucker können aber manuell per IP hinzugefügt werden.
# Netzwerk-Modus prüfen:
docker inspect filamenthub | grep NetworkMode
# Muss sein: "NetworkMode": "host"

Windows — Lokal

  1. Python 3.10+ installieren (python.org)
  2. Repository klonen: git clone https://github.com/d3nn3s08/FilamentHub.git
  3. Virtuelle Umgebung: python -m venv venv dann venv\Scripts\activate
  4. Pakete: pip install -r requirements.txt
  5. .env erstellen: copy .env.example .env
  6. Datenbank: python -m alembic upgrade head
  7. Starten: python run.py → http://localhost:8085

Erste Konfiguration (.env)

ADMIN_PASSWORD_HASH=$2b$12$...   # bcrypt Hash (optional)
ADMIN_COOKIE_SECURE=false         # true nur für HTTPS
APP_VERSION=Beta v1.6 · FilamentHub
FILAMENTHUB_DB_PATH=/app/data/filamenthub.db

🚀 Erste Schritte

In ~10 Minuten startklar
  1. Drucker hinzufügen — Druckerverwaltung → "Drucker suchen" (Scanner) oder manuell mit IP + Seriennummer + Zugangscode
  2. Verbindung prüfen — Grüner Punkt = verbunden. Rot = Fehlerbehandlung
  3. Material anlegen — Spulen Hersteller → "Material hinzufügen" (Typ, Farbe, Leergewicht)
  4. Erste Spule anlegen — Spulen Verwaltung → "Neue Spule" → Material wählen, Gewicht eintragen
  5. AMS-Slot zuweisen (falls vorhanden) — AMS Übersicht → Slot anklicken → Spule wählen
  6. Druck starten — FilamentHub erkennt automatisch: Job wird angelegt, Verbrauch getrackt

🖨️ Druckerverwaltung

Drucker-Scanner

FilamentHub scannt das LAN per TCP-Port-Scanning — kein mDNS, kein Broadcast. Bambu Ports: 6000, 8883, 990. Klipper: 7125.

Docker: Scanner braucht network_mode: host. Im Bridge-Modus sieht der Container nur 172.17.x.x statt dein LAN.
Scan-TypBeschreibungDauer
Quick ScanBekannte IPs + ±5 Nachbarn~2s
Network ScanVollständiger CIDR-Sweep (z.B. /24)~30s
Bambu DetectNur Bambu-Ports~5s
Klipper DetectNur Port 7125~3s

Verbrauch nachtragen

  1. Druckaufträge → Job anklicken
  2. "Verbrauch eintragen"
  3. Spule(n) auswählen und Gramm eingeben
  4. "Speichern"

Verbrauchsberechnung

Verbrauch [g] = Gewicht vor Druck − Gewicht nach Druck
Kosten        = (Verbrauch / Spulengewicht) × Spulenpreis
Beispiel:       (50g / 1000g) × 20€ = 1,00€

  2. Admin Panel → "Bambu Cloud" → E-Mail und API-Key eingeben

🔧 Klipper & MMU (Happy Hare)

Klipper / Moonraker einrichten

  1. Druckerverwaltung → "Drucker hinzufügen" → Typ: Klipper
  2. IP-Adresse + Port 7125 eingeben
  3. API-Token eingeben falls Moonraker Auth aktiv ist
  4. "Verbindung testen" → "Speichern"

Happy Hare MMU einrichten

  1. MMU-Klipper (Seitenleiste) öffnen
  2. Klipper-Drucker auswählen
  3. Anzahl Gates eingeben
  4. Jeden Gate: Spule aus Datenbank zuweisen
  5. "Speichern"

🔒 Sicherheit

Im Heimnetz reicht die Standardkonfiguration. HTTPS ist nur nötig wenn FilamentHub von außen erreichbar sein soll.

nginx Reverse Proxy (HTTPS)

server {
    listen 443 ssl;
    server_name filamenthub.deine-domain.de;
    ssl_certificate     /etc/letsencrypt/live/deine-domain.de/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/deine-domain.de/privkey.pem;
    location / {
        proxy_pass http://localhost:8085;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
# Danach in .env: ADMIN_COOKIE_SECURE=true

Empfohlene Netzwerkstruktur

Heimnetz (192.168.1.x) FilamentHub-Server ──────────────────── TCP Port 8883/6000 ──► Dein PC / Handy │ ▼ IoT-VLAN (192.168.10.x) Bambu Drucker Sonstige Smart-Home-Geräte Router-Regel: Heimnetz → IoT: ERLAUBT IoT → Heimnetz: BLOCKIERT
FilamentHub hat keine API-Auth für normale Endpunkte. Bei öffentlichem Zugriff: Reverse Proxy mit HTTP-Basisauth vorschalten.

🛠️ Fehlerbehandlung

Drucker-Scanner findet nichts

Häufigste Ursache bei Docker: Container läuft im Bridge-Modus (172.17.x.x) statt host-Netzwerk.
# Netzwerk-Modus prüfen
docker inspect filamenthub | grep NetworkMode
# Muss sein: "NetworkMode": "host"

# Korrigieren und neu starten:
docker compose down && docker compose up -d

Weitere Ursachen:

  • Drucker in anderem Subnetz → Network Scan mit manuellem CIDR starten
  • Bambu "Privacy Mode" → Drucker-Display → Einstellungen → Lokales MQTT aktivieren
  • Firewall blockiert Port 8883/6000 → nc -zv DRUCKER_IP 8883 testen

Drucker verbindet nicht (bleibt rot)

  • IP-Adresse korrekt?
  • Zugangscode korrekt? (Drucker-Display → Einstellungen → WLAN)
  • Seriennummer korrekt? (Aufkleber unten am Gerät)
  • Drucker und FilamentHub im gleichen Netzwerk?

❓ FAQ — Häufige Fragen

Funktioniert FilamentHub ohne Internet?

Ja. MQTT läuft lokal im LAN. Bambu Cloud ist optional.

Kann ich mehrere Drucker gleichzeitig nutzen?

Ja. Beliebig viele — Bambu, Klipper und manuelle Drucker können gemischt werden.

Muss ich für jeden Druck manuell eintragen?

Nein. Bei MQTT-Verbindung wird alles automatisch getrackt.

Warum muss network_mode: host gesetzt sein?

Im Bridge-Modus sieht der Container nur das Docker-interne Netz (172.17.x.x), nicht dein Heimnetz. Mit host-Modus teilt er den Netzwerk-Stack des Hosts.

Geht FilamentHub auf Synology ohne SSH?

Eingeschränkt. Der Scanner funktioniert nicht (kein host-Mode im GUI). Drucker können aber manuell per IP hinzugefügt werden.

Kann ich FilamentHub auf Raspberry Pi betreiben?

Ja. Das Docker-Image ist ARM-kompatibel (python:3.12-slim-bookworm).

Wie genau ist die AMS-Gewichtsmessung?

Frische Spulen: ±5%. Fast leere Spulen: ±20%. Für präzise Werte: Spule wiegen und manuell eintragen.

Warum zeigt ein Job "0g Verbrauch"?

MQTT war unterbrochen oder Spule war nicht zugewiesen. Lösung: Druckaufträge → Job → "Verbrauch eintragen".

Ist Bambu Cloud Pflicht?

Nein. FilamentHub funktioniert vollständig ohne Cloud.

Unterschied AMS und AMS Lite?

AMS Full: X1C, P1S, P1P — bis zu 16 Slots. AMS Lite: A1, A1 Mini — 4 Slots, vereinfachte Hardware.

Wo liegt die Datenbank?

Docker: /app/data/filamenthub.db (im gemappten Volume). Windows: Pfad aus FILAMENTHUB_DB_PATH in .env.

Backup ohne Unterbrechung?

docker exec filamenthub sqlite3 /app/data/filamenthub.db ".backup /app/data/backup.db"
